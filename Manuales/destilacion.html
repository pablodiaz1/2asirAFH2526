Destilación de una IA Manual tipo presentación
1/9

¿Qué es la destilación de modelos?

Técnica para transferir el conocimiento de un modelo grande y preciso (profesor) a un modelo más pequeño (alumno), manteniendo rendimiento con menos recursos.

Idea clave

El alumno intenta imitar las salidas del profesor. Así se reduce tamaño/consumo sin perder demasiado rendimiento.

Profesor: grande, preciso Alumno: ligero, eficiente Objetivo: rendimiento aceptable

1) Crear el entorno de trabajo

Creamos la carpeta del proyecto y entramos dentro para trabajar.

Comandos

mkdir destilacion_ia
cd destilacion_ia

Dentro de esta carpeta irán los scripts: probar, evaluar, destilar, etc.

2) Crear y ejecutar probar.py

Script de prueba para verificar que el entorno y el modelo responden.

Comandos

nano probar.py
python probar.py

3) Crear y ejecutar evaluar.py

Evaluación del rendimiento (métricas) del modelo alumno.

Comandos

nano evaluar.py
python evaluar.py

4) Crear y ejecutar evaluar_profesor.py

Evaluación del modelo profesor para compararlo con el alumno.

Comandos

nano evaluar_profesor.py
python evaluar_profesor.py

5) Crear y ejecutar kl_divergencia.py

Cálculo de divergencia KL para medir diferencia entre distribuciones (profesor vs alumno).

Comandos

nano kl_divergencia.py
python kl_divergencia.py

Cuanto menor sea la divergencia, más se parece el alumno al profesor.

6) Ejecutar el proceso de destilación

Lanzamos el script principal que realiza el entrenamiento/destilación.

Comando

python destilar.py
Entrada: profesor + datos Salida: alumno destilado

7) Prueba final con acertijos

Creamos un archivo de prueba práctica para comparar respuestas del profesor y del alumno.

Comandos

nano acertijos.py
python acertijos.py

Compara aciertos y calidad de respuesta entre ambos modelos.

Conclusión

El profesor suele ser más preciso, pero el alumno puede imitar su comportamiento con un tamaño muy inferior.

Resumen de resultados

El profesor consigue mejores métricas (accuracy y F1-score) y acierta más acertijos. Aun así, el alumno mantiene un rendimiento aceptable con una reducción de parámetros muy grande.

Profesor: mayor precisión Alumno: rendimiento aceptable Reducción: > 15× parámetros
